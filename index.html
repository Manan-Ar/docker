Objective

This project demonstrates how a web application can be containerized using Docker. It focuses on isolating services, enabling communication between them, and simplifying deployment.

Technology Stack

  • Node.js (Backend)
  • PostgreSQL (Database)
  • Docker
  • Docker Compose
  • Macvlan / Ipvlan Networking

Architecture

Client → Backend Container → PostgreSQL Container

Each component runs in a separate container. Communication is handled through Docker networking, while persistent data is stored using volumes.

API Endpoints

  • POST /data
  • GET /data
  • GET /health
app.post('/data', ...); app.get('/data', ...); app.get('/health', ...);

Docker Implementation

  • Multi-stage build used to optimize image size
  • Alpine-based image for lightweight containers
  • Non-root user improves security
FROM node:18-alpine AS builder WORKDIR /app COPY package*.json ./ RUN npm install COPY . . FROM node:18-alpine WORKDIR /app COPY --from=builder /app . CMD ["node","server.js"]

Docker Compose

version: '3' services: backend: build: ./backend db: build: ./database volumes: - pgdata:/var/lib/postgresql/data volumes: pgdata:

Networking

Macvlan networking assigns unique IP addresses to each container, allowing them to behave like independent systems on the same network.

docker network create -d macvlan \ --subnet=192.168.1.0/24 \ --gateway=192.168.1.1 \ -o parent=eth0 mynet

Note: Host cannot directly communicate with Macvlan containers.

Report Summary

This project involved building a containerized web application using Docker, Node.js, and PostgreSQL. The backend and database were deployed in separate containers and managed using Docker Compose. A multi-stage build was used to optimize the image size and improve performance.

Macvlan networking was implemented to assign separate IP addresses to containers, allowing them to function as independent systems. Persistent storage was achieved using Docker volumes, ensuring that data remains intact even after container restarts.

The project helped in understanding key DevOps concepts such as containerization, networking, and service orchestration in a practical environment.
