Overview

This project demonstrates containerization of a web application using Docker. Multiple services are isolated into containers and connected using Docker networking, ensuring scalability, portability, and ease of deployment.

Technology Stack

  • Node.js Backend
  • PostgreSQL Database
  • Docker
  • Docker Compose

Architecture

Client → Backend Container → Database Container

API

  • POST /data
  • GET /data
  • GET /health

Docker

FROM node:18-alpine
WORKDIR /app
COPY . .
RUN npm install
CMD ["node","server.js"]

Docker Compose

version: "3"
services:
  backend:
    build: .
    ports:
      - "5000:5000"
    depends_on:
      - db

  db:
    image: postgres
    environment:
      POSTGRES_DB: appdb
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:

Networking

Macvlan networking allows containers to have their own IP addresses and behave like independent systems.

Key Learning

  • Service isolation

  • Docker networking

Report Summary

This project focuses on building a containerized web application using Docker, Node.js, and PostgreSQL. The backend and database are deployed in separate containers and managed using Docker Compose, which simplifies multi-service deployment.

A multi-stage Docker build was implemented to optimize the final image size by removing unnecessary dependencies. This improves performance and reduces resource usage.

Macvlan networking was used to assign individual IP addresses to containers, allowing them to behave like independent systems on the same network. This also highlights differences compared to bridge networking.

To ensure data persistence, Docker volumes were used for the PostgreSQL database, so data remains intact even after container restarts.

Overall, the project demonstrates practical understanding of containerization, service orchestration, networking, and deployment in a DevOps environment.
